# AI短视频创作平台 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品背景
基于 MiniMax 强大的视频生成大模型能力（包括 Video Generation、Video Agent、Video Prompt等技术），打造一款面向短视频创作者、自媒体人和小说推文创作者的 AIGC 创作平台。解决创作者在视频制作中**角色一致性难保持**、**场景连贯性差**以及**长文本转视频人工拆解效率低下**的痛点。

### 1.2 核心目标
提供一站式的“故事提取 -> 分镜拆分 -> 资产匹配 -> 视频生成”工作流。通过高度模块化的角色和背景管理，结合智能的文本拆解引擎，实现高效、高质量、风格统一的短视频自动化生成。

---

## 2. 核心功能描述

### 2.1 角色管理模块 (Character Management)
**功能描述**：用于创建、编辑和管理短视频中出现的各类角色资产，确保视频生成时的角色一致性。
- **角色创建与设定**：
  - 支持上传角色形象参考图（Reference Image）。
  - 支持输入角色名称、外貌特征、服装风格、性格设定等描述信息。
- **角色库资产化**：可视化的角色卡片列表，支持角色的增删改查、标签分类。
- **自动化上下文绑定**：
  - 在后续的故事处理中，一旦系统识别或用户指定了出场角色，系统将自动把该角色的形象参考和 Prompt 参数打包，用于填充视频生成 API 的上下文信息。

### 2.2 故事背景管理模块 (Story Background Management)
**功能描述**：用于集中管理短视频故事发生的环境、场景和氛围资产。
- **背景创建与设定**：
  - 支持上传场景参考图（如赛博朋克街道、中式古代宫廷、现代都市咖啡馆等）。
  - 支持输入场景环境、光影氛围、镜头风格等文本 Prompt。
- **背景库资产化**：以缩略图形式展示的背景库，方便用户快速浏览和选用。
- **动态调用**：在生成视频分镜时，作为环境上下文信息动态插入到最终的视频生成提示词中。

### 2.3 章节拆分模块 (Chapter Splitting)
**功能描述**：将用户输入的长篇故事/脚本，智能解析并拆分为多个适合生成短视频画面的分镜片段（Segments）。
- **长文本输入与解析**：提供富文本编辑器，供用户粘贴或输入长篇故事内容。
- **AI 智能拆解**：
  - 接入大语言模型能力，根据文本的情节发展、场景转换、动作变化，自动将长故事切割成合理的短片段。
- **分镜编辑与人工微调**：
  - 以列表形式展示拆分后的片段。
  - 支持用户手动合并、拆分片段，或修改片段的文本描述，确保分镜逻辑和画面感合理。
- **智能实体识别**：自动高亮片段文本中出现的“角色名”，并与【角色管理】中的角色库进行自动映射匹配。

### 2.4 片段视频生成模块 (Video Generation Workstation)
**功能描述**：短视频创作的核心工作台。基于拆分好的片段，结合角色和背景设定，一键调用大模型生成视频。
- **交互方式 (右键触发)**：
  - 在分镜列表中，用户可选中某个片段并**点击右键**，在弹出的菜单中选择“生成视频”。
- **背景选择机制**：
  - 触发生成前，系统弹出配置确认框，允许用户从【故事背景管理】库中为当前片段**选择或切换背景**。
- **自动化 Prompt 融合**：
  - 系统自动提取片段中的**角色设定** + 用户选择的**背景设定** + **片段动作/情节**。
  - 遵循 MiniMax `video-prompt` 最佳实践公式（主体 + 环境 + 动作 + 镜头语言），在后台自动拼装出最优的生成提示词。
- **调用模型生成**：
  - 通过接入 MiniMax `Video Generation` 或基于模板的 `Video Agent` 接口执行生成任务。
- **结果反馈与重绘**：生成完毕后在片段旁展示视频预览，支持下载、保存，或调整 Prompt 后重新生成。

## 3. 用户操作主流程 (User Workflow)

1. **资产准备阶段**：
   - 用户进入【角色管理】添加男女主角及配角的外貌设定和参考图。
   - 用户进入【背景管理】添加故事将要发生的几个主要场景设定。
2. **故事导入与拆分**：
   - 用户在工作台输入一段长篇故事（例如：一段修仙小说情节）。
   - 点击“智能拆分章节”，系统将其切分为 10 个连续的分镜片段，并自动标记每个片段中出场的角色。
3. **视频生成阶段**：
   - 用户在片段 1 上点击右键，系统提示配置。
   - 用户选择背景为“宗门大殿”，系统自动提取主角信息，点击生成。
   - 依次或批量完成其余片段的视频生成。
4. **预览与导出**：
   - 在时间轴或列表中连贯预览所有生成的短视频片段，最终导出完整视频。

## 4. 架构与技术选型建议

### 4.1 六边形架构设计 (Hexagonal Architecture)
为了保证系统的高可扩展性和灵活性，考虑到未来需要接入多个视频生成大模型平台（如 MiniMax、OpenAI Sora、Runway、可灵等），系统架构要求采用**六边形架构（端口与适配器模式，Ports and Adapters）**：
- **核心领域层 (Domain)**：抽象并定义视频生成、片段拆分、资产管理等业务逻辑的通用接口（Ports），核心业务不依赖于具体的第三方 API 实现。
- **适配器层 (Adapters)**：
  - **入站适配器 (Inbound Adapters)**：处理前端界面或外部服务发来的 API/WebSocket 请求。
  - **出站适配器 (Outbound API Adapters)**：针对不同的底层 AI 视频生成平台实现具体的适配器。例如，为 MiniMax 提供 `MiniMaxVideoAdapter`，在其中处理特有的参数封装与鉴权。当业务需要支持新的视频生成平台时，只需添加新的适配器类并注入到核心接口中，从而实现无缝切换或多平台并发调用。

### 4.2 视频生成接口与策略建议 (以 MiniMax 为例)
1. **统一视频生成接口 (Video Generation API)**：
   - 在核心层调用统一接口，底层由具体 Adapter 将合并后的完整 Prompt 和参考图传递给对应的平台进行文生视频 / 图生视频生成。
2. **视频 Agent 与模板 (Video Agent Templates)**：
   - 针对提供预设模板的平台（如 MiniMax `video-agent`），封装模板匹配策略，为用户提供一键套用风格特效的能力，简化 Prompt 编写难度。
3. **统一 Prompt 工程优化 (Prompt Engineering)**：
   - 在领域层封装统一的 Prompt 改写服务。由于长故事的片段通常是文学化表达，该服务首先调用 LLM 将其转化为视频模型友好的**客观画面描述语言**（如“一个年轻男子，特写镜头，面部表情愤怒，眉头紧锁”），然后再交由底层 Adapter 适配不同平台的具体指令格式，从而提升各平台的成片成功率。
